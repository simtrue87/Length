# 폰 실기기 검증 체크리스트

> 사용 기기:
> 일자:
> 빌드: `flutter build apk --release` → `build/app/outputs/flutter-apk/app-release.apk`

## 0. 설치 & 첫 부팅
- [ ] 디바이스 ABI 확인: `adb shell getprop ro.product.cpu.abi` → 보통 `arm64-v8a`
- [ ] 해당 ABI 빌드 설치:
  - `adb install -r build/app/outputs/flutter-apk/app-arm64-v8a-release.apk` (대부분)
  - 또는 `app-armeabi-v7a-release.apk` (구형 32-bit)
- [ ] 앱 아이콘·이름 "Length"로 보임
- [ ] 첫 부팅 → 모드 선택 화면 정상 표시
- [ ] 설치 APK 크기 ~45~56MB 범위 (opencv 포함)

## 1. 사진 모드 4종 핵심 흐름

### 1.0 호모그래피 정확도 — 사다리꼴 카드 케이스 (전 모드 공통)
- [ ] 카드를 평면 위에 두고 **카메라를 일부러 비스듬히** 기울여 촬영 (15~30°)
- [ ] 사진에서 카드가 사다리꼴로 보임 확인
- [ ] 참조 객체 모드 → 4점 보정 → 카드 가로 변(85.6mm)을 두 점 탭으로 측정
- [ ] 결과가 **84~87mm** 안에 들어오면 호모그래피 정상 (이전에는 90mm+ 또는 80mm 미만이 나올 수 있었음)
- [ ] 같은 사진에서 다른 평면 위치(카드와 떨어진 곳) 측정해도 정확

### 1.1 참조 객체 (신용카드) — opencv 자동 감지 포함
- [ ] 모드 선택 → "참조 객체" (추천 카드 강조 확인)
- [ ] 갤러리에서 카드 사진 선택
- [ ] **자동 감지 성공 케이스**: 평평한 단색 배경 + 정면 촬영 → "자동 감지됨" 메시지 + 4점이 카드 모서리에 정렬
- [ ] **감지 실패 케이스**: 어두운 사진 또는 카드와 배경 대비 낮은 사진 → "자동 감지 실패" + 기본 폴백 사각형
- [ ] 자동 감지 후 핸들 미세 조정 가능 여부
- [ ] 빗각(약 30°) 촬영 시 감지 동작 (실패해도 폴백으로 흐름 유지)
- [ ] 조명 변화(그림자/역광) 시 감지 성공률 체감
- [ ] "다음" → 측정 종류 칩 (거리/둘레/면적/각도)
- [ ] 거리 2점, 결과 화면 진입
- [ ] **자동 감지 → 신뢰도 "정확"**, **수동 폴백 → 신뢰도 "참고용"** 확인
- [ ] 메모: 자동 감지 시 "자동 4점 감지", 수동 시 "수동 4점 보정"
- [ ] 결과에 사진+측정선 미리보기 + 값 표시
- [ ] 자이로 SnackBar (단말 기울일 때 표시되는지)

### 1.2 QR/마커 (자동 감지)
- [ ] PC에서 QR 인쇄(예: 50mm × 50mm)
- [ ] 측정 대상 옆에 QR 두고 촬영
- [ ] 모드 → "QR/마커"
- [ ] QR 한 변 mm 입력값이 마지막 사용값으로 자동 채워짐 (M 영속화)
- [ ] "갤러리/촬영" 진입 → 감지 메시지 표시
- [ ] 자동 4점이 QR 모서리에 정렬
- [ ] 감지 실패 케이스: 흐릿한 사진으로 시도 → 수동 4점 폴백
- [ ] 측정 후 결과 라벨에 QR 크기 표기

### 1.3 캘리브레이션 + 평면
- [ ] 모드 → "캘리브레이션 + 평면"
- [ ] 높이·FOV 마지막 값 자동 채움 (M 영속화)
- [ ] 단말 평평히 들고 기울기 표시가 0°에 근접
- [ ] 일부러 기울이면 카드 색상 errorContainer로 변경
- [ ] 위에서 수직으로 촬영 (테이블 위 책 등)
- [ ] 측정 후 결과 메모에 `h=XXXmm FOV XX°` 표기
- [ ] 기울기 > 20° 상태로 측정 시 신뢰도 `낮음` 강등 확인

### 1.4 AI 깊이 추정 (TFLite 인프라 + 스텁 폴백)
- [ ] 모드 → "AI 깊이 추정"
- [ ] 첫 화면 안내 문구 확인:
  - 모델 미배포: "모델 미배포 — 스텁으로 흐름 검증."
  - 모델 배포 후: "Depth Anything V2 모델 로드 완료. 결과는 상대 깊이로 참고용입니다."
- [ ] 임의 사진 → 진행률 표시 → 측정 단계
- [ ] 결과 메모에 "⚠ 상대 깊이(스텁)" 또는 "⚠ 상대 깊이(Depth Anything V2)" 노트
- [ ] 신뢰도 `낮음` 배지
- [ ] (모델 배포 후) 모드 라벨이 "사진 — AI 깊이"(스텁 표기 없음)로 변경

## 2. 결과·이력·공유·설정

### 2.1 결과 화면
- [ ] 단위 토글 mm/cm/inch (각도 모드는 토글 없음, ° 표시)
- [ ] 단위 선택이 영속화: 다른 측정 진입 시에도 같은 단위 (J)
- [ ] "이력에 저장" → SnackBar → 버튼 비활성
- [ ] 공유 버튼 → 시스템 시트에 사진+텍스트 동시 전송

### 2.2 이력 화면
- [ ] 도구 섹션 → "이력 보기"
- [ ] 필터 칩(전체/거리/둘레/면적/각도) 동작
- [ ] 날짜 그룹 헤더(오늘/어제/이번주/이전)
- [ ] 썸네일 표시 (이미지 있는 경우)
- [ ] 없는 경우 종류별 아이콘
- [ ] 스와이프 좌측 → 삭제
- [ ] 항목 탭 → 결과 화면 재진입

### 2.3 영속성 (앱 종료·재실행)
- [ ] 앱 완전 종료(스와이프 닫기)
- [ ] 재실행 → 이력 그대로
- [ ] 단위·캘리브레이션·QR 기본값도 보존

## 3. 권한 & 뒤로가기

### 3.1 카메라 권한
- [ ] 사진 모드 → "촬영" 첫 시도 → 시스템 권한 요청
- [ ] 거부 → SnackBar "카메라 권한이 필요합니다."
- [ ] 다시 시도 → 다시 요청
- [ ] "다시 묻지 않기" 거부 → "촬영" 시도 시 다이얼로그 "카메라 권한 필요" + "설정 열기" 버튼
- [ ] "설정 열기" → 시스템 설정 진입

### 3.2 뒤로가기 보호
- [ ] 사진 모드 진입 → 이미지 선택 → 보정/측정 단계
- [ ] 시스템 뒤로가기 → "측정을 종료할까요?" 다이얼로그
- [ ] "계속" → 화면 유지
- [ ] "종료" → 모드 선택으로 복귀
- [ ] 초기 입력 화면에서 뒤로 → 다이얼로그 없이 바로 복귀
